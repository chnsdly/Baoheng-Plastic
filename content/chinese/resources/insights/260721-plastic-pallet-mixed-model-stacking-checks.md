当两个塑料托盘型号外观看起来很接近时，仓库团队可能会在收货、清洗、回收分拣或卸车时把它们叠在一起。这个临时做法会带来实际风险。不同的面板高度、脚型、底部接触区、套叠结构、损坏角位和承载数据，可能让看似整齐的堆垛变成倾斜、卡滞或不稳定的堆垛。

如果现场同时使用多个塑料托盘型号、颜色、供应商版本、客户循环托盘或回收托盘系列，可以用这份检查来控制风险。实用规则很简单：只有确认属于同一兼容型号且状态合格的托盘，才按正常堆垛处理。混合或身份不确定的堆垛，应先暂扣，直到型号、路线和堆垛限制得到确认。

这是一份仓库控制指南，不是托盘型号批准书，也不是法律规则。最终限制仍取决于实际托盘设计、状态、载荷、搬运设备、储存区域、供应商数据和当地安全要求。

## 为什么混合型号堆垛需要单独规则

堆高指南回答的是已知托盘和已知载荷能堆多高。来料检验计划回答的是一批托盘能否放行。颜色管理系统帮助区分不同区域。混合型号堆垛位于这些控制之间：它要判断已经进入现场的托盘能不能互相叠放。

官方安全资料支持优先确认稳定性。[OSHA 29 CFR 1910.176](https://www.osha.gov/laws-regs/regulations/standardnumber/1910/1910.176) 涉及物料安全储存以及通道、过道保持畅通。OSHA 的[机动工业车辆载荷搬运指南](https://www.osha.gov/etools/powered-industrial-trucks/operating-forklift/load-handling)要求操作人员只搬运稳定或已安全布置的载荷。[HSE 托盘安全指南](https://www.hse.gov.uk/pubns/pm15.htm)把托盘安全视为采购、使用、检查、搬运和培训共同控制的问题。

这些资料不会批准任何具体塑料托盘型号。它们支持一个操作边界：如果混合堆垛不能确认稳定、兼容并且便于检查，就不要把它当作正常储存。

## 先定义什么是同型号

"蓝色托盘"或"1200 x 1000 mm 托盘"并不足够。同型号规则应使用会影响堆垛接触和搬运的特征。

| 检查项 | 为什么重要 |
| --- | --- |
| 型号和供应商身份 | 相似尺寸可能掩盖面板、脚部或底部结构差异。 |
| 模具或版本 | 设计更新可能改变套叠、底部接触、筋位高度或加固方式。 |
| 尺寸和高度 | 细小高度差可能集中压力，并让空托盘堆倾斜。 |
| 底部类型 | 九脚、川字、田字、双面和套叠托盘的堆放方式不同。 |
| 状态 | 破角、弯曲底梁、磨损脚位或翘曲面板，会让本来兼容的托盘变得不安全。 |
| 路线状态 | 食品区、隔离、客户专用、出口或回收托盘，即使型号相同，也可能有不同放行规则。 |

如果其中一项未知，就把堆垛标记为混合或不确定。这并不表示每个托盘都要报废，而是表示应先暂扣、分拣或降级使用，再进入正常流转。

更完整的收货控制可连接到[塑料托盘来料检验计划](/zh/resources/insights/2617-plastic-pallet-incoming-inspection-plan/)。区域身份控制可与[塑料托盘颜色分区指南](/zh/resources/insights/2619-plastic-pallet-color-coding-warehouse-segregation/)保持一致。

## 混合堆垛常见四类失效

### 1. 空托盘卡住或倾斜

空托盘堆通常搭建很快。套叠托盘可能卡在不匹配的脚型上。田字或双面托盘可能压在凸起面板结构上，而不是设计接触点上。中间夹入一个略高的托盘，就可能让上方整堆倾斜。

把空托盘堆当作受控储存，而不是无风险的现场整理。按型号定义最大空盘堆高，并要求操作人员在托盘坐不平、分离困难或出现明显倾斜时停止放行。

### 2. 载货堆垛把压力传到错误接触点

载货地堆风险更高，因为最下面的托盘承受整堆竖向压力。如果上方托盘的脚或底梁不同，压力可能落在下方托盘的薄弱面板区域。即使两个托盘单独使用都合格，下方托盘也可能变形。

已知型号和已知载荷可使用[塑料托盘堆高规划指南](/zh/resources/insights/2622-plastic-pallet-stack-height-floor-storage-planning/)。当堆垛包含混合或不确定托盘身份时，应使用本页的暂扣和分拣规则。

### 3. 叉车搬运变得不可预测

混合堆垛在举升时可能移位，因为叉孔、底部接触和堆垛对齐并不一致。操作人员也可能为了补偿而斜插，或举升一个已经倾斜的堆垛。OSHA 机动工业车辆指南支持只搬运稳定或已安全布置的载荷，但它不能替代现场自己的托盘堆垛规则。

放行混合堆垛前，应确认堆垛在举升、转弯、下降以及经过正常地面路线时保持平稳。如果堆垛摇晃、卡住或散开，应按型号分拣。

### 4. 状态控制丢失

混合堆垛会破坏卫生、客户或隔离控制。已清洗食品区托盘可能与未验证回收托盘叠放。客户专用循环托盘可能流入通用库存。被拒收的托盘可能被藏在合格堆垛中间。

当型号身份和状态不一致时，应停止移动并分拣托盘。不要把颜色相同当作状态相同的证据。

## 暂扣、分拣、放行表

使用一个简短决策表，让收货、清洗、仓储和叉车团队不用长时间调查也能执行。

| 堆垛发现 | 状态 | 下一步 |
| --- | --- | --- |
| 同型号、受控时同版本、同路线状态、无可见损坏 | 按定义的空盘或载货堆垛限制放行 | 在需要记录的操作中记录型号和堆垛限制。 |
| 同型号但损坏状态混杂 | 暂扣或分拣 | 放行前移除损坏、翘曲、开裂、污染或不稳定托盘。 |
| 尺寸和颜色相同，但型号、底部或版本不同 | 按型号分拣 | 未经供应商或现场负责人确认兼容前，不作为正常堆垛使用。 |
| 未知回收托盘与已放行托盘混放 | 隔离或回收分拣 | 放行前确认既往用途、清洗状态、损坏情况和客户循环身份。 |
| 载货混合堆垛或明显倾斜 | 除受控卸垛外停止移动 | 在主管控制下拆垛，并检查底部托盘和载货状态。 |
| 托盘卡滞、套叠困难或分离不顺 | 暂扣做兼容性复核 | 把它当作堆垛设计问题，而不只是操作问题。 |

目标不是拖慢每一次移动，而是在混合堆垛变成储存、叉车、卫生或客户问题之前，让异常显性化。

## 向供应商确认什么

对于重复采购或多场地项目，应让供应商明确说明堆垛兼容性：

- 准确型号、尺寸、底部类型、图纸或版本；
- 当前生产批次是否能与早期已供版本一起堆放；
- 最大空盘堆高和套叠方向，如适用；
- 载货托盘是否只能同型号叠放；
- 哪些区域不应承受不同底部结构形成的点压；
- 磨损脚位、开裂底梁、翘曲面板或损坏套叠结构的检查点；
- 模具、底部结构、材料或高度变化影响堆垛时的变更通知。

RFQ 中可以写：

> 供应商应说明所报托盘型号、底部结构、堆叠或套叠行为、可兼容的早期版本以及空盘和载货堆放限制。除非供应商数据和现场测试确认买方路线下的稳定接触，否则不批准混合型号堆垛。

如果托盘选择本身仍未确定，可先比较[套叠托盘与货架托盘的选择规则](/zh/resources/insights/2606-nestable-vs-rackable-plastic-pallets/)，再决定是否让一个型号覆盖所有堆垛场景。

## 正常放行前的现场测试

在允许新托盘型号与现有流转托盘混用前，做一次简短测试。

1. 按计划高度建立一个同型号空盘堆。
2. 用新旧型号建立最可能发生的意外混合堆。
3. 检查每个托盘是否压在设计接触点上。
4. 举升、转弯、短距离行驶、下降并分离堆垛。
5. 如果存在清洗或低温条件，重复相关条件后的检查。
6. 检查摇晃、倾斜、卡滞、脚位损坏、面板压痕和分离风险。
7. 用简单语言记录放行规则：仅同型号、可与指定型号兼容、仅限空盘、禁止载货混堆，或待供应商确认。

对于载货堆垛，还要加入真实重量、产品占板面积、停放时间和地面条件。不要因为空盘混堆看起来可接受，就批准载货混堆。

## 给买方和仓库团队的决策规则

只有在型号身份、堆垛接触、托盘状态、路线状态和堆高都清楚时，才放行塑料托盘堆垛。

托盘颜色或尺寸相同，但型号、底部类型、版本或路线状态不同，应分拣。

托盘卡住、倾斜、摇晃、损坏、既往用途不明，或隐藏隔离和客户专用状态时，应暂扣。

允许混合型号载货堆垛、自动化搬运、食品区回收、客户循环复用或高空盘堆放前，应由供应商和现场负责人确认。

清楚的同型号堆垛规则是一项小控制。它可以减少不稳定空盘堆、隐藏损坏托盘、客户循环丢失，以及因假设相似塑料托盘性能相同而产生的争议。
